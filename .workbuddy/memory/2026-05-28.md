# 2026-05-28 工作日志

## 落墨页真机交互失效修复

### 问题
真机测试中：所有按钮（文字/纸张/排版/保存）失效，仅输入框可操作；输入文字后Canvas无渲染响应。

### 根因
1. **底部区域被TabBar遮挡**：`.bottom-area` 的 z-index(100001) 与 `custom-tab-bar`(99998) 冲突，真机上 Canvas 2D 原生组件优先级更高，拦截了工具栏按钮的触摸事件
2. **字体加载阻塞渲染链**：`_doRender` 中 `await loadFont()` 无超时保护，云开发未初始化时 loadFont 重试 3 次 x 30 秒 = 最多 90 秒阻塞，期间所有渲染请求无法执行
3. **渲染请求未去重**：多个 `_triggerRender` 并发调用 `_doRender`，导致多个渲染 Promise 并行
4. **`JSON.parse(JSON.stringify())` 深拷贝问题**：会丢失 undefined 属性值

### 修复内容
- luomo.wxss：bottom-area z-index 提升至 100002，弹窗 z-index 提升至 100003；工具栏高度 56rpx→68rpx，按钮触摸热区增大；canvas-area padding-bottom 计算；添加 pointer-events: auto
- luomo.wxml：canvas-wrapper 的 bindtouchmove 改为 catchtouchmove，阻止触摸事件穿透到 Canvas 原生层
- luomo.js：_doRender 中字体加载添加 8 秒独立超时保护（Promise.race），超时后使用 getFallbackFont() 继续渲染；添加 _renderingInProgress 渲染锁和 _renderRequested 去重机制；_getEffectiveTemplate 改用 _safeDeepClone 替代 JSON.parse(JSON.stringify())；onShow 中渲染延迟 100ms
- renderer.js：_scaleTemplateForDPR 改用 _safeDeepClone；添加 _safeDeepClone 工具函数
