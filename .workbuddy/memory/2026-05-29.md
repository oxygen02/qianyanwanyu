# 第二轮修复 — 真机卡死根因分析（13:11）

## 用户反馈
"仍然是载入非常之慢，慢到像是死机了停服了，慢到点击任何按钮都无反应"

## 根因定位

### 致命瓶颈 #1：generatePaperTexture 像素级 SimplexNoise
- 原代码对 Canvas 每个像素调用 3 次 noise2D
- DPR=2 画布 750×1000 = 75万像素 × 3 = **225万次噪声计算**
- DPR=3 设备上可达 500万+ 次，**完全同步阻塞 JS 主线程**
- 还有 8 层叠加效果 （grooves/micrograin/vignette/edges/curl...）

### 致命瓶颈 #2：JSON.parse(JSON.stringify()) 深度克隆
- `_getEffectiveTemplate` 和 `_scaleTemplateForDPR` 各调用一次
- 每次渲染触发两次深度克隆，对象包含嵌套的 layout/paper/ink/font/decoration

### 致命瓶颈 #3：50ms 输入防抖
- 用户快速打字时，每 50ms 触发一次完整渲染管线

## 修复方案

1. **noise 降采样 stride=2/3**（paper.js）：4-9x 加速，75万→19万 noise 调用
2. **首屏立即显示纯色**（luomo.js `_renderEmptyTemplate`）：先 fillRect 纯色（<1ms），纹理用 setTimeout(0) 异步生成
3. **输入防抖 50→200ms**（luomo.js）：避免渲染风暴
4. **DPR 上限 2**（luomo.js `_initCanvas`）：高 DPR 设备像素数减半
5. **浅拷贝替代 JSON 深度克隆**（luomo.js `_getEffectiveTemplate`, renderer.js `_scaleTemplateForDPR`）：约 50x 快

## 文件变更
- miniprogram/engine/paper.js（noise stride 降采样）
- miniprogram/engine/renderer.js（浅拷贝替代 JSON 克隆）
- miniprogram/pages/luomo/luomo.js（首屏纯色、DPR上限、防抖、浅拷贝）
