## 修复两个排版渲染 bug

### Bug #1：数字字间距异常（`33` 渲染为 `3 3`）
- **根因**：`typesetter.js` justify 两端对齐分支，当一行字符数很少（如 2 个数字）时，gap 被均分到整行宽度，导致字间距极大
- **修复**：`typesetter.js` 第 226 行，justify 条件加了两个约束：`lineChars.length >= 4`（少于4字不触发 justify），且 `gap > fontSize * 0.5` 时回退到 letterSpacing（不撑开）

### Bug #2：换行文字不显示（第二段落消失）
- **根因**：`luomo.js` 的 settings 中 `paragraphSpacing: 25`（UI 滑块值 0-100），被直接赋值给 `tpl.layout.paragraphSpacing`，而 `typesetter.js` 里是 `lineInCol += paragraphSpacing`（加行数），25 行远超 `linesPerColumn`，直接触发换页，第二段被推到第 1 页（pageIndex=1），用户停留在第 0 页看不到
- **修复1**：`luomo.js` `_getEffectiveTemplate` 里对 paragraphSpacing 换算：`(s.paragraphSpacing / 100) * 2`，即 25 → 0.5 行
- **修复2**：`typesetter.js` paragraphSpacing 取值加 `Math.min(..., 3)` 上限保护，防止异常大值再次触发此问题

### 修改的文件
- `miniprogram/engine/typesetter.js`
- `miniprogram/pages/luomo/luomo.js`

## 三轮修复（截图反馈：数字/行间距/段落间距三个问题）

### 根因重构：`_getEffectiveTemplate` 执行顺序错误
- `_applyLayoutMode` 在所有用户设置（lineHeight/letterSpacing/textAlign/indent）之后执行，**把这些设置全覆盖成模式默认值**
- 之前只把 paragraphSpacing 移到了后面，但其他同行间距相关的设置仍然被覆盖

### 修复 1：将所有用户排版覆盖移到 `_applyLayoutMode` 之后
- **文件**：`luomo.js` `_getEffectiveTemplate`
- `s.lineHeight`、`s.letterSpacing`、`s.textAlign`、`s.firstLineIndent` 都移到 `_applyLayoutMode` 之后
- 这解释了为什么用户调行间距滑块无效——每次渲染都被模式默认值覆盖

### 修复 2：数字字间距——ASCII 半角→全角转换
- **根因**：数字"3"是 ASCII 半角字符，canvas fillText 渲染宽度约 20px，但 typesetter 按 fontSize=40px 分配位置，多出 20px 空隙
- **方案**：`typesetter.js` 增加 `toFullWidth()` 函数，排版前将 0-9/A-Z/a-z 转为全角 CJK 字符（U+FF10-FF19/U+FF21-FF3A/U+FF41-FF5A），使数字占满全角单元格
- 这符合传统中文印刷规范

### 修复 3：扩大滑块范围
| 滑块 | 旧范围 | 新范围 | 说明 |
|------|--------|--------|------|
| 行间距 | 80-300 (0.8x-3.0x) | 80-400 (0.8x-4.0x) | WXML max + JS clamp |
| 字间距 | -5-25 (-5%~25%) | -10-40 (-10%~40%) | WXML + JS clamp |
| 段落间距 | 0-50 → 0-1行 | 0-100 → 0-4行 | WXML max + 映射因子 *4 |

### 附带修复
- `letterSpacing` 上限 cap：`fontSize * 0.15` → `fontSize * 0.35`（配合新40%范围）
- `onLetterSpacingStep` 补上缺失的 `s.letterSpacing` 赋值（原只设了 val）
- 段落间距标签从 "px" 改为无单位显示
- `onLayoutSettingsReset` 补回 `settings.paragraphSpacing` 重置

## 修复字间距无效 + 扩大负值范围

用户反馈：字间距调到最小（-20%）仍然间距很大，输入数字间距更大，失去纸感书写效果。

### 根因分析

1. **字间距滑块最小值 -20% 不够**：等宽中文字体（思源宋体）每个字符占 1em 宽度，即使字间距 -20%，字符之间仍有 0.8em 的间距，视觉上仍然松散
2. **DPR 重复缩放防护**：`_scaleTemplateForDPR` 添加 `_dprScaled` 标记，防止 `estimatePageCount` 和 `renderPage` 对同一模板重复缩放

### 修复内容

1. **typesetter.js**：letterSpacing 计算改为双向限制 `Math.max(-fontSize, Math.min(..., fontSize))`，支持 -100%~100%
2. **luomo.wxml**：字间距 slider min 从 -20 改为 **-100**
3. **luomo.js**：`onLetterSpacingStep` clamp 从 -20~100 改为 **-100~100**
4. **renderer.js**：`_scaleTemplateForDPR` 添加 `_dprScaled` 标记防重复缩放

### 使用效果

- 字间距 0%：标准等宽排版（1em 字进宽）
- 字间距 -50%：字符间距减半（0.5em）
- 字间距 -100%：字符完全紧贴（0 间距，可能重叠）

用户可根据需要自由调节，达到纸感手写的紧凑效果。
