## 一、需求文档分析报告

### 1.1 现有文档（trae.md）的优势

| 维度 | 评价 |
|------|------|
| 信息架构 | 清晰完整，三Tab（落墨/拾文/吾卷）命名有文学气息 |
| 技术深度 | 墨迹8种特效算法、版心参数、Perlin噪声纹理——可直接编码 |
| 合规分析 | 个人主体限制梳理彻底，无遗漏风险 |
| 模板体系 | 10套模板参数精确（RGB/噪点/行距），覆盖主要用户场景 |
| 开发计划 | Phase划分合理，单人可执行 |

### 1.2 现有文档的不足与缺口

#### 缺口一：交互哲学未落地成具体规范

文档提到"触控唤醒、墨散动画"但缺乏**精确的交互状态机**定义：
- 边缘热区唤醒 → 工具浮层出现 → 4秒自动消失 → 这个流程的每个状态转换缺乏确切定义
- "墨水散在水中，浓到淡到消失"是诗意描述，缺乏CSS/Canvas动画规格（时长、缓动曲线、触发条件）

#### 缺口二：书页视觉质感未达到"印刷感"标准

现有引擎（engine/renderer.js）存在关键问题：
- `drawText()` 直接用 `ctx.fillText()` 渲染，字是"贴在纸上"的，未实现"印在纸上"的感官
- 缺少**纸张纤维与墨迹的物理交互**：字符下方的纸张应该有因墨水渗透产生的微弱晕圈
- `applyFiberTexture()` 使用 `ctx.putImageData()` 会覆盖底色，逻辑有潜在问题
- 缺少**画布光影层**：真实书页在偏角光照下有微弱的立体感和光泽差异

#### 缺口三：输入与输出区布局比例设计不符合"极简"要求

用户明确要求：**输出展示区占极大篇幅**，功能隐藏在边缘，首次用户看到后消失。  
现有 `luomo.wxml` 布局是：canvas区（42-52vh）+ 输入区（48-58vh），比例颠倒了。  
应该是：canvas占75-80vh，输入区折叠在底部，只露出一行高度，点击展开。

#### 缺口四：字体加载策略不完整

`loadCustomFont()` 的云存储路径是硬编码占位符（`cloud://qianyan-env.xxx`），  
字体分包策略（哪些字体内置、哪些CDN按需加载）未定义。

#### 缺口五：分页与多页阅读体验未设计

`getPageText()` 用简单字数分割，未考虑：
- 段落不被截断（需要按段落换行处理）
- 翻页动画（PTQBookPageView移植计划存在但未启动）
- 多页时的导航状态（左滑/右滑手势）

#### 缺口六：拾文（文章转书）功能边界模糊

个人主体小程序网络请求受限（只能请求已配置域名），文章抓取云函数的域名白名单策略未定义，  
导致"支持任意网页"的承诺在技术上无法直接实现。

### 1.3 各来源文档优缺点对照

| 来源 | 独特贡献 | 不足 |
|------|----------|------|
| CatPaw主文档 | 框架完整、Prompt体系、合规清单 | 交互细节不够精确，动效描述诗意化而非技术化 |
| Kimi思考 | 商业模式（个人版已裁剪）、印章阳刻算法、品牌视觉锤(①圆圈)概念 | 过多企业版功能，需大量裁剪 |

## 二、产品定位（终版）

### 2.1 核心命题

> 把冰冷键盘输出的刻板文字，转化成好像在读一本真实的书——字**印**在纸上，不是**贴**在纸上。

### 2.2 感官标准（设计判断基准）

用户看到生成结果时，第一感官应该是：

1. **这是纸**：看到纸张纤维纹理、岁月黄斑、边缘微毛边
2. **这是墨**：字的边缘有微弱晕染，个别笔画有飞白，有铅字缺角感
3. **这是书**：有版心留白、页码、有时代感的排版节奏

不应该是：
- 一张图片上有字（贴图感）
- 白纸黑字加了个滤镜

### 2.3 目标用户（终版）

| 层级 | 年龄 | 核心驱动力 |
|------|------|------------|
| 核心 | 18-30岁 | 社交分享欲：把自己写的文字变成"有质感的内容" |
| 次要 | 30-45岁 | 仪式感：读书笔记、金句收藏、给文字赋予重量 |
| 长尾 | 45岁+ | 复古情感：怀旧的书卷气 |

## 四、核心交互规范（终版）

### 4.1 落墨页布局比例

```
┌──────────────────────────┐
│  书页Canvas展示区          │  占页面高度 75-80%
│  （占绝对主体篇幅）         │
│  首次进入：显示空书页        │
│  有内容：实时渲染文字        │
│                          │
│  页码浮标：右下角淡显        │
└──────────────────────────┘
│  底部输入条（折叠态）        │  约 80rpx 高
│  "落笔..."  [字数] [⬆]      │  轻触展开输入
└──────────────────────────┘
```

**输入区展开态**（键盘弹起时）：
- canvas区自适应收缩至约 50vh（仍占主体）
- 输入框展开，显示 3-5 行
- 底部保留操作栏（存入 / 清空）

### 4.2 边缘唤醒工具栏

**交互状态机：**

```
[静息态] → 触碰四边热区(各30px宽) → [工具浮现态]
                                        ↓
                                   墨散动画展开（400ms）
                                        ↓
                                   [工具可见态] → 4秒无操作 → [工具消散态]
                                        ↓                           ↓
                                   点击工具项                  墨散反向消失(500ms)
                                        ↓
                                   [面板弹出态]
```

**首次进入特殊逻辑：**
1. 进入页面后 1.5秒，自动显示工具栏一次（教育用户）
2. 3秒后自动消散
3. 此后只响应边缘触碰，不再自动显示

**墨散动画规格：**
```css
/* 工具栏出现：墨滴落水扩散 */
出现：scale(0.3) opacity(0) → scale(1.05) opacity(1) → scale(1) opacity(1)
时长：400ms，easing: cubic-bezier(0.34, 1.56, 0.64, 1)（弹性）

/* 工具栏消失：墨水扩散淡出 */
消失：scale(1) opacity(1) → scale(1.1) opacity(0)
时长：500ms，easing: cubic-bezier(0.4, 0, 1, 1)（加速）
```

### 4.3 工具浮层内容

浮层为圆角胶囊形，居中浮于书页上方（不遮挡书页主体内容）：

```
[ 模板 | 纸张 | 字体 | 排版 | 注释 | 印章 | 导出 ]
```

点击任一项 → Bottom Sheet 从底部滑入（300ms，cubic-bezier(.32,.72,0,1)）

### 4.4 书页实时渲染规范

**防抖策略：**
- 拼音输入中（候选词未确认）：不触发渲染
- 候选词确认/直接输入：50ms防抖后触发
- 参数调整（字体/纸张等）：立即全量重绘

**渲染优先级：**
1. 纸张底层（仅配置变化时重绘，输入文字时跳过）
2. 增量文字（只重绘最后几行，不重绘整页）
3. 装饰层（印章/水印，仅配置变化时叠加）

## 五、视觉质感规范（7大渲染层）

### 5.1 Layer 1：纸张质感层

**目标效果**：底色 + 纤维纹理 + 岁月渐变 + 污渍斑点 + 边缘效果

**纸张纤维纹理修正方案**（修复现有代码问题）：

```javascript
// 问题：直接 putImageData 会清除底色
// 正解：先 fillRect 底色，再用 globalCompositeOperation: 'multiply' 叠加纹理
applyFiberTexture(ctx, width, height, paperRGB, intensity) {
  // 创建离屏纹理
  const offscreen = wx.createOffscreenCanvas({ width, height })
  const offCtx = offscreen.getContext('2d')
  
  // 在离屏canvas绘制纤维
  const imageData = offCtx.createImageData(width, height)
  // ... Perlin噪声逻辑 ...
  offCtx.putImageData(imageData, 0, 0)
  
  // 以multiply模式叠加到主canvas（不破坏底色，产生真实纹理感）
  ctx.save()
  ctx.globalCompositeOperation = 'multiply'
  ctx.drawImage(offscreen, 0, 0)
  ctx.restore()
}
```

**岁月感强化**：
- 四角加深（径向渐变，从中心透明到边角深褐色 rgba(120,100,80,0.12)）
- 随机3-8个不规则污渍斑（椭圆形，模糊半径2-5px，透明度0.08-0.15）
- 顶边/底边微弱灰色渐变（模拟阅读磨损）

### 5.2 Layer 2：油墨印刷层

**核心命题**：字不是贴在纸上，是**印**在纸上。

**印刷质感的技术实现**：

```javascript
renderCharWithInkPenetration(ctx, char, x, y, fontConfig, paperFiber) {
  const inkColor = fontConfig.color || [42, 42, 42]
  const fontSize = fontConfig.size || 16
  
  // 第一道：渗透晕圈（在纸张纤维中扩散的墨水）
  ctx.save()
  ctx.globalCompositeOperation = 'multiply'
  ctx.shadowBlur = 1.5 + paperFiber * 1.0  // 纤维越粗，渗透越多
  ctx.shadowColor = `rgba(${inkColor[0]}, ${inkColor[1]}, ${inkColor[2]}, 0.25)`
  ctx.fillStyle = `rgba(${inkColor[0]}, ${inkColor[1]}, ${inkColor[2]}, 0.08)`
  ctx.fillText(char, x, y)
  ctx.restore()
  
  // 第二道：主体墨迹（油墨实体，深沉有重量感）
  ctx.save()
  ctx.globalCompositeOperation = 'multiply'
  const inkVar = (Math.random() - 0.5) * config.inkVarRange
  const finalColor = adjustColor(inkColor, inkVar)
  ctx.fillStyle = `rgb(${finalColor.join(',')})`
  ctx.fillText(char, x + jitter.x, y + jitter.y)
  ctx.restore()
  
  // 第三道：高光留白（模拟铅字凸起处油墨稀薄的高光）
  if (Math.random() < 0.15) {
    ctx.save()
    ctx.globalCompositeOperation = 'screen'
    ctx.fillStyle = `rgba(255,255,255,0.06)`
    ctx.fillText(char, x + jitter.x - 0.3, y + jitter.y - 0.3)
    ctx.restore()
  }
}
```

**关键参数**：
- `globalCompositeOperation: 'multiply'` 是实现"印在纸上"而非"贴在纸上"的核心
- multiply模式：纸张纤维纹理 × 墨迹颜色 = 字迹随纸张纹理起伏变化
- 渗透晕圈半径与纸张纤维值联动（纤维粗 → 晕染大，如宣纸 vs 胶版纸）

### 5.3 Layer 3：排版布局层

**版心参数（专业出版级）**：

### 5.5 Layer 5：光影立体层

**书页微立体感**（营造厚度感，不是3D效果）：

```

    0 8px 24px rgba(0,0,0,0.07),  // 散射阴影
    0 1px 0 rgba(255,255,255,0.6) inset;  // 顶边高光（模拟页面顶边）
```

**页面内部光照**（Canvas内）：
- 顶部左侧：微弱高光（透明白色渐变，透明度0.03-0.05）
- 底部右侧：微弱阴影（深褐渐变，透明度0.04-0.06）
- 效果：营造自然光从左上照射的书页感

## 七、功能规范

### 7.1 落墨（文字成书）核心流程

```
用户进入落墨页
    ↓
展示空书页（现代散文模板，完整纸张质感渲染）
    ↓
1.5秒后：工具浮层自动出现（墨散动画），3秒后自动消散
    ↓
用户触碰书页 → 键盘弹起 → 输入区展开
    ↓
每个字确认 → 50ms防抖 → Canvas增量重绘（multiply模式渲染）
    ↓
达到单页上限 → 自动分页，提示"下一页"
    ↓
多页时：左右滑动翻页（PTQBookPageView动画）
    ↓
点击边缘 → 工具浮层出现 → 选择导出 → 导出中心页
```

### 7.2 拾文（文章转书）

**输入路径**：

| 路径 | 场景 | 技术方案 |
|------|------|----------|
| A: 粘贴链接 | 普通网页 | 云函数 + axios + Readability |
| B: 手动粘贴 | 微信公众号/任意文本 | 前端 textarea + 智能清洗 |

**域名白名单策略**（个人主体版现实约束）：
- 路径A仅支持已在微信后台配置域名的网站
- 初期配置：知乎(zhihu.com)、豆瓣(douban.com)、简书(jianshu.com)等常用内容平台
- 微信公众号：必须用路径B手动粘贴，检测到公众号链接时弹窗友好提示

**文本清洗管道**（路径B）：
```
粘贴内容 → 去除HTML标签 → 关键词黑名单过滤
    → 段落智能分割（连续换行合并）
    → 去除广告/引导关注语句
    → 输出结构化段落数组
    → 预览界面确认
    → 进入排版引擎
```

### 7.3 导出功能

| 格式 | 技术方案 | 说明 |
|------|----------|------|
| 图片 PNG | wx.canvasToTempFilePath | 单页 |
| 长图 | 多页Canvas竖向拼接 | 完整内容一图 |

现有代码用 ctx.fillText() 直接绘制，字是浮在纸面上的。

第一道（渗透晕圈）：shadowBlur 联动纸张纤维值，globalCompositeOperation:'multiply'，opacity 0.08

## 纸张纹理修正
现有 putImageData 方式会清除底色。
正解：在离屏Canvas生成纹理，再以 multiply 模式叠加到主Canvas。

请实现完整的 BookPageRenderer 升级版，保持与现有 engine/*.js 文件的接口兼容。
